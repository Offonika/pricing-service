# S1.1 — Создать каркас приложения FastAPI

## Цель
Построить базовую структуру FastAPI-приложения с healthcheck и конфигами.

## Что нужно сделать
- Создать каталоги `app/api`, `app/core`, `app/models`, `app/schemas`, `app/services`, `app/workers`.
- Добавить точку входа `app/main.py` с эндпоинтом `/health` (200 OK, например `{"status": "ok"}`).
- Реализовать конфигурацию через Pydantic Settings (`app/core/config.py`), читать переменные окружения для БД, Redis и др.
- Обновить `README.md` секцией «Как запустить» (локально/через Docker, если готов).

## Артефакты/файлы
- `app/main.py`, `app/api/__init__.py` и роутер healthcheck.
- `app/core/config.py`
- Обновлённый `README.md`

## Definition of Done
- FastAPI поднимается локально, `/health` возвращает 200.
- Конфиг через Pydantic Settings, значения берутся из env.
- Структура каталогов соответствует `docs/architecture.md` и `docs/constitution.md`.
