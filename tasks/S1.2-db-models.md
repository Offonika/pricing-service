# S1.2 — Определить базовые модели БД

## Цель
Добавить первичные доменные модели и связи в SQLAlchemy для MVP.

## Что нужно сделать
- Создать модели:
  - `Product` (SKU, название, бренд, категория, статусы, флаги ABC/XYZ, активность).
  - `Competitor` (название, сайт, активность).
  - `CompetitorPrice` (product_id, competitor_id, цена, наличие, дата сбора).
- Настроить `relationship` и базовые индексы (product_id + competitor_id + date).
- Подготовить модели к использованию в Alembic (метадата/registry).
- Обновить/дополнить Pydantic-схемы, если нужны для тестов/эндпоинтов.

## Артефакты/файлы
- `app/models/*.py`
- При необходимости `app/schemas/*.py`

## Definition of Done
- Модели описаны, импортируются без ошибок.
- Связи и индексы определены.
- Есть базовые тесты создания/чтения моделей (или фикстуры для S1.6).
- Готово к генерации первой миграции (S1.3).
