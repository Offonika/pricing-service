Задача: поддержать несколько выгрузок ZenLogs с одинаковой структурой (green, amigo, poiskzip-moba, …) и ежедневно писать их в общий каталог/историю.

1. **Конфиг/Env**  
   - В `app/core/config.py` добавить список источников (например, `zenlogs_sources: list[str]`).  
   - В `.env.example`/`README` описать формат `ZENLOGS_SOURCES=moba:https://...,green:https://...`.  
   - Поддержать опции таймаута/verify общими для всех источников.

2. **Парсер/импорт**  
   - Обобщить `app/services/importers/zenlogs_moba.py`, чтобы функция принимала `competitor_name` и URL.  
   - При чтении XLSX использовать общий маппинг колонок (структура файлов одинаковая).  
   - Убедиться, что `CompetitorCatalogRecord.competitor` задаётся для каждого источника.

3. **Job/CLI**  
   - В `app/workers/zenlogs.py` пройтись по списку источников, для каждого вызывать скачивание/парсинг/апсерт.  
   - Логировать статистику по каждому конкуренту и общий итог.  
   - CLI `tasks/import_zenlogs_competitors.py` остаётся прежним.

4. **Тесты**  
   - Юнит-тест, который эмулирует две ссылки и проверяет, что job создаёт записи для обоих конкурентов без дублей.  
   - Проверка обработки пустых/недоступных файлов (job не падает целиком).

5. **Документация**  
   - `docs/PRD.md`/`docs/architecture.md`: описать мульти-источник ZenLogs, формат переменной `ZENLOGS_SOURCES`, ежедневный импорт.  
   - README: обновить раздел переменных окружения и инструкцию по запуску job.
